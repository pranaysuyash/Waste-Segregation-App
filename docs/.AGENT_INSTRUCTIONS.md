# Instructions for AI Coding Agents

## 🚨 CRITICAL: Read Before Any Code Changes

### Mandatory First Step

**ALWAYS read `docs/APP_KNOWLEDGE_BASE.md` before making ANY changes to this repository.**

This is not optional. The knowledge base contains:

- What actually exists vs. what's aspirational (e.g., on-device models are placeholder-only)
- Critical architecture details (state management, inference pipeline, services)
- All screens, models, and UI flows documented
- Environment requirements (`.env` setup, Firebase config)
- Known issues and troubleshooting (blank screen debugging, etc.)
- Update protocols (how to keep docs current)

### Your Workflow

1. **Before coding:**
   - Read `docs/APP_KNOWLEDGE_BASE.md` (mandatory).
   - Check `CHANGELOG.md` for recent changes.
   - Review `docs/status/CURRENT_ISSUES_SUMMARY.md` for known blockers.
   - Verify `.env` file exists with required API keys.

2. **During coding:**
   - Consult knowledge base sections relevant to your work.
   - Follow existing patterns (Riverpod, MVVM, Hive storage).
   - Log appropriately using `WasteAppLogger`.
   - Maintain type safety and null safety.

3. **After coding:**
   - Update `docs/APP_KNOWLEDGE_BASE.md` if you:
     - Added/changed models or inference logic.
     - Modified major UI flows or screens.
     - Changed environment requirements.
     - Fixed significant bugs.
   - Update "Last verified" date in knowledge base.
   - Add entry to `CHANGELOG.md` if user-facing.
   - Document any new services/providers/viewmodels in knowledge base.

### Common Pitfalls to Avoid

1. **Don't assume on-device inference works** - it's a placeholder returning dummy data.
2. **Don't commit `.env` file** - it contains secrets and is in `.gitignore`.
3. **Don't skip knowledge base updates** - next agent will waste time rediscovering your changes.
4. **Don't use print()** - use `debugPrint()` or `WasteAppLogger` methods.
5. **Don't ignore null safety** - app uses sound null safety, maintain it.
6. **Don't add dependencies without documenting** - update knowledge base with new packages.

### When in Doubt

1. Search knowledge base for relevant section.
2. Check CHANGELOG for similar past work.
3. Review existing code patterns in same area.
4. Ask user for clarification rather than guessing.
5. Document assumptions in code comments.
